## Why Mural exists

There is no good home for wallpapers on a Mac. System Settings buries them, and most wallpaper apps are paid, web-wrapped, or full of stock art you will never use.

Mural is a small native app with one job: keep the wallpapers you actually like, still or moving, and put them on your screen. No accounts, no storefront, no web wrapper.

## Mural Studio

**When nothing in the box fits, sketch your own.** A small wallpaper workshop built into the sidebar. Start from a drawn backdrop or a photo of your own, then tune it until it feels right.

- ✓ &nbsp;**Twelve drawn backdrop styles** — dunes, aurora, plaid, moonrise, and friends.
- ✓ &nbsp;**Seventeen curated palettes**, free-form colours, and a variation dice.
- ✓ &nbsp;Everything renders locally at **2880×1800** — one click from your desktop.

## Get Mural

**Make it yours.** Grab the app, or build it from source in a couple of minutes. Either way — free, native, and no strings attached.

<p align="center">
  <a href="https://github.com/FrederickStempfle/Mural/releases/download/v1.0.0/Mural.dmg"><strong>↓&nbsp; Download Mural.dmg</strong></a>
</p>

```sh
# or build it yourself
git clone https://github.com/FrederickStempfle/Mural.git
cd Mural && ./scripts/build-app.sh
open dist/Mural.app
```

The build script creates an ad-hoc signed `dist/Mural.app` containing the Mural wallpaper extension. To package it as an installer, run `./scripts/build-dmg.sh`, which produces `dist/Mural.dmg`. To run the library straight from source without packaging:

```sh
swift run Mural
```

<sub>Requires macOS 14 or later. Animated video wallpapers need macOS 26.</sub>

## Good to know

<details>
<summary><b>macOS says Mural “can’t be opened”. Now what?</b></summary>
<br />

That's Gatekeeper being cautious about apps downloaded outside the App Store. **Right-click Mural.app and choose Open**, or go to **System Settings → Privacy & Security** and click **Open Anyway**. You only have to do this once — and since Mural is open source, you can always build it from source instead.

</details>

<details>
<summary><b>Is it really free? What's the catch?</b></summary>
<br />

Really free, no catch. Mural is open source, has no accounts, no storefront, and no telemetry to sell — everything renders locally on your Mac.

</details>

<details>
<summary><b>Which Macs does it run on?</b></summary>
<br />

Any Mac on **macOS 14 (Sonoma) or later**. Animated video wallpapers need **macOS 26** and Xcode 26; stills and the Studio work everywhere.

</details>

<details>
<summary><b>How do animated wallpapers actually work?</b></summary>
<br />

Mural uses Apple's private `WallpaperExtensionKit` integration: `WallpaperAgent` keeps playback running on the Desktop and Lock Screen after the app closes, across displays and Spaces.

If macOS cannot apply the video automatically, Mural opens the **Mural — Video Wallpapers** collection in System Settings so you can select it there. When a video is removed from Mural, its deployed system copy is retained while macOS still uses it, so an active wallpaper never loses its media.

</details>

<details>
<summary><b>My video wallpaper stopped after a macOS update.</b></summary>
<br />

The video-wallpaper integration leans on private Apple frameworks, so a macOS update can occasionally knock it over. Check the [latest release](https://github.com/FrederickStempfle/Mural/releases) — fixes usually land there first. Still wallpapers are never affected.

</details>
